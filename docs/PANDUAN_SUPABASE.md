### 1. Apakah Gratis? **YA.**
Supabase memiliki **Free Tier** (Paket Gratis) yang sangat dermawan dan lebih dari cukup untuk website organisasi seperti HMI Cabang Garut.
- **Database**: 500MB (Cukup untuk menampung ribuan berita/artikel text).
- **Project**: Hingga 2 project aktif.
- **Bandwidth**: 5GB per bulan.
- **Penyimpanan File/Gambar**: 1GB.
**Kesimpulan**: Anda tidak perlu membayar sepeserpun selama pengguna website belum jutaan per bulan.

### 2. Apakah Aman? **YA.**
- **Basis Teknologi**: Dibangun di atas **PostgreSQL**, database open-source terpopuler dan terkuat di dunia.
- **Enkripsi**: Data Anda dienkripsi saat disimpan dan saat dikirim (SSL).
- **Proteksi Akses**: Menggunakan sistem **RLS (Row Level Security)**. Kita bisa mengatur agar *siapapun bisa membaca berita*, tapi *hanya admin yang bisa menulis/mengedit*.

### Langkah 1: Buat Akun & Project
1. Buka [https://supabase.com](https://supabase.com).
2. Klik tombol **"Start your project"**.
3. Login menggunakan akun **GitHub** (Rekomendasi) atau Email.
4. Klik **"New Project"**.
   - **Name**: `hmicabanggarut`
   - **Database Password**: Buat password kuat (Simpan baik-baik, jangan sampai lupa!).
   - **Region**: Pilih `Singapore (SG)` agar akses cepat dari Indonesia.
   - Klik **"Create new project"**.
   - Tunggu sekitar 1-2 menit hingga proses setup selesai.

### Langkah 2: Ambil Kunci Rahasia (API Keys)
Setelah project aktif:
1. Di dashboard Supabase, lihat menu kiri paling bawah, klik icon **Settings** (Roda Gigi).
2. Pilih menu **API**.
3. Anda akan melihat bagian **Project URL** dan **Project API keys**.
4. Salin **Project URL**.
5. Salin **`anon`** `public` Key.
   > âš ï¸ **PENTING**: Jangan pernah membagikan `service_role` key. Cukup yang `anon` saja.

### Langkah 3: Masukkan ke Website Kita
1. Kembali ke VS Code.
2. Buat file baru bernama `.env.local` (titik env titik local) di folder paling luar.
3. Isi file tersebut dengan data yang tadi disalin:
   ```env
   NEXT_PUBLIC_SUPABASE_URL=paste_project_url_disini
   NEXT_PUBLIC_SUPABASE_ANON_KEY=paste_anon_key_disini
   ```

### Langkah 4: Membuat Tabel (Database)
Kita tidak perlu coding manual, kita pakai fitur **SQL Editor**.
1. Di dashboard Supabase, cari icon **SQL Editor** di menu kiri (gambar terminal `>_`).
2. Klik **New Query**.
3. Copy kode di bawah ini dan Paste di kotak query Supabase:

```sql
-- TABEL BERITA
create table news (
  id bigint generated by default as identity primary key,
  title text not null,
  slug text unique not null,
  date date not null default CURRENT_DATE,
  image_url text,
  content text,
  excerpt text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- TABEL EVENT
create table events (
  id bigint generated by default as identity primary key,
  title text not null,
  slug text unique not null,
  date date not null,
  location text,
  image_url text,
  description text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- AKTIFKAN FITUR KEAMANAN (RLS)
alter table news enable row level security;
alter table events enable row level security;

-- ATURAN AKSES: Semua orang boleh BACA (Select)
create policy "Public can view news" on news for select using (true);
create policy "Public can view events" on events for select using (true);

-- ATURAN AKSES: Sementara buka akses tulis untuk Development (Nanti kita kunci setelah fitur login jadi)
create policy "Public can insert news" on news for insert with check (true);
create policy "Public can update news" on news for update using (true);
create policy "Public can delete news" on news for delete using (true);
create policy "Public can insert events" on events for insert with check (true);
create policy "Public can update events" on events for update using (true);
create policy "Public can delete events" on events for delete using (true);
```
4. Klik tombol **Run** (Kanan bawah atau atas). Jika sukses, akan muncul pesan "Success" dan tidak ada error.

### Langkah 5: Mengatur Penyimpanan Gambar
1. Di menu kiri, klik icon **Storage** (Gambar Ember).
2. Klik **"New Bucket"**.
3. Beri nama: `images`.
4. Pastikan saklar **"Public Bucket"** dalam posisi **ON** (Aktif). Ini wajib agar gambar bisa dilihat pengunjung website.
5. Klik **Save**.
